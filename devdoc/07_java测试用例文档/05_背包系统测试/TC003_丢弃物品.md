# TC003: 丢弃物品

## 测试目标
验证玩家能够成功丢弃背包中的物品，包括消耗品、装备等。

## 测试环境
- 服务器状态：正常运行
- 客户端：Java测试客户端
- 网络环境：本地网络

## 测试步骤

### 步骤1: 建立TCP连接
- 客户端与服务器建立TCP连接
- 验证连接成功

### 步骤2: 构造登录请求
- 创建 `REQ_LOGIN` 对象
- 设置 `openid`、`type`、`token`、`platID`、`version`、`clientIP` 等字段

### 步骤3: 编码登录请求
- 使用 `MessageCodec` 编码登录请求
- 验证编码成功

### 步骤4: 发送登录请求
- 向服务器发送编码后的登录请求
- 验证发送成功

### 步骤5: 接收登录响应
- 接收服务器返回的登录响应
- 验证响应数据长度正确

### 步骤6: 解码登录响应
- 使用 `MessageCodec` 解码登录响应
- 验证解码成功，响应类型为 `RES_LOGIN`

### 步骤7: 验证登录成功
- 检查 `RES_LOGIN` 对象中的 `error` 字段是否为 `null`
- 检查 `RES_LOGIN` 对象中的 `authkey` 字段是否不为 `null`
- 验证登录成功

### 步骤8: 构造丢弃物品请求
- 创建丢弃物品请求对象
- 设置物品ID、物品数量等相关参数

### 步骤9: 编码丢弃物品请求
- 使用 `MessageCodec` 编码丢弃物品请求
- 验证编码成功

### 步骤10: 发送丢弃物品请求
- 向服务器发送编码后的丢弃物品请求
- 验证发送成功

### 步骤11: 接收丢弃物品响应
- 接收服务器返回的丢弃物品响应
- 验证响应数据长度正确

### 步骤12: 验证丢弃物品响应
- 使用 `MessageCodec` 解码丢弃物品响应
- 验证解码成功，响应类型正确
- 检查响应中的丢弃结果是否正确
- 验证丢弃物品测试通过

## 预期结果
- 登录成功
- 丢弃物品请求发送成功
- 收到正确的丢弃物品响应
- 物品丢弃成功，背包中的物品数量减少
