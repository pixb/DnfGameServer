# TC008_成就解锁通知

## 测试用例信息

- **测试用例ID**: TC008
- **测试用例名称**: 成就解锁通知
- **测试类型**: 功能测试
- **优先级**: 中
- **关联模块**: 成就系统
- **测试环境**:
  - JDK 17+
  - Maven 3.6+
  - MySQL 8.0+
  - 游戏服务器运行中

## 测试目标

验证当玩家完成成就时，能够收到成就解锁的通知消息，包括成就名称、描述和奖励等信息。

## 测试前置条件

1. 游戏服务器已启动并正常运行
2. 玩家已成功登录游戏
3. 玩家已选择角色进入游戏
4. 玩家有接近完成的成就

## 测试数据

| 字段名 | 类型 | 值 | 说明 |
|-------|------|-----|------|
| openid | String | test_openid_008 | 测试用OpenID |
| version | String | 1.0.0 | 客户端版本号 |
| clientIP | String | 127.0.0.1 | 客户端IP地址 |
| roleGuid | String | role_guid_008 | 角色GUID |
| achievementId | int | 1004 | 测试用成就ID |
| progress | int | 10 | 测试用成就进度（达到目标） |
| targetProgress | int | 10 | 成就目标进度 |

## 测试步骤

1. **连接服务器**
   - 建立与游戏服务器的TCP连接
   - 验证连接是否成功

2. **发送登录请求**
   - 构造REQ_LOGIN消息
   - 发送登录请求到服务器
   - 接收并解析登录响应
   - 验证登录是否成功

3. **发送获取角色列表请求**
   - 构造REQ_GET_ROLE_LIST消息
   - 发送请求到服务器
   - 接收并解析角色列表响应
   - 验证是否获取到角色列表

4. **发送选择角色请求**
   - 构造REQ_SELECT_ROLE消息
   - 发送请求到服务器
   - 接收并解析选择角色响应
   - 验证是否成功选择角色

5. **发送成就进度更新请求**
   - 构造REQ_UPDATE_ACHIEVEMENT_PROGRESS消息
   - 设置成就ID为测试数据中的achievementId
   - 设置进度为测试数据中的progress（达到目标）
   - 发送请求到服务器
   - 接收并解析成就进度更新响应

6. **验证成就进度更新响应**
   - 验证响应是否成功
   - 验证成就是否已完成

7. **接收成就解锁通知**
   - 等待并接收服务器推送的成就解锁通知
   - 解析成就解锁通知消息

8. **验证成就解锁通知**
   - 验证通知是否包含成就解锁信息
   - 验证通知中的成就ID是否与完成的成就一致
   - 验证通知中的成就名称、描述是否正确
   - 验证通知中的奖励信息是否正确

9. **断开连接**
   - 关闭与服务器的TCP连接

## 预期结果

1. 服务器返回成就进度更新成功响应
2. 成就状态更新为已完成
3. 服务器推送成就解锁通知
4. 成就解锁通知包含完整的成就信息：
   - 成就ID
   - 成就名称
   - 成就描述
   - 成就等级
   - 奖励列表
   - 解锁时间

## 测试异常处理

1. **服务器未运行**
   - 预期：连接失败，测试终止
   - 处理：启动服务器后重新测试

2. **登录失败**
   - 预期：登录响应失败，测试终止
   - 处理：检查登录参数后重新测试

3. **选择角色失败**
   - 预期：选择角色响应失败，测试终止
   - 处理：检查角色GUID后重新测试

4. **成就进度更新失败**
   - 预期：响应失败，测试终止
   - 处理：检查成就ID和进度后重新测试

5. **未收到成就解锁通知**
   - 预期：测试失败
   - 处理：检查服务器配置和网络连接后重新测试

6. **成就解锁通知信息不完整**
   - 预期：测试失败
   - 处理：检查服务器实现后重新测试
