# TC003_获取成就详情

## 测试用例信息

- **测试用例ID**: TC003
- **测试用例名称**: 获取成就详情
- **测试类型**: 功能测试
- **优先级**: 高
- **关联模块**: 成就系统
- **测试环境**:
  - JDK 17+
  - Maven 3.6+
  - MySQL 8.0+
  - 游戏服务器运行中

## 测试目标

验证玩家能够成功获取单个成就的详细信息，包括成就的完整属性、进度和奖励信息。

## 测试前置条件

1. 游戏服务器已启动并正常运行
2. 玩家已成功登录游戏
3. 玩家已选择角色进入游戏
4. 玩家已有成就数据

## 测试数据

| 字段名 | 类型 | 值 | 说明 |
|-------|------|-----|------|
| openid | String | test_openid_003 | 测试用OpenID |
| version | String | 1.0.0 | 客户端版本号 |
| clientIP | String | 127.0.0.1 | 客户端IP地址 |
| roleGuid | String | role_guid_003 | 角色GUID |
| achievementId | int | 1001 | 测试用成就ID |

## 测试步骤

1. **连接服务器**
   - 建立与游戏服务器的TCP连接
   - 验证连接是否成功

2. **发送登录请求**
   - 构造REQ_LOGIN消息
   - 发送登录请求到服务器
   - 接收并解析登录响应
   - 验证登录是否成功

3. **发送获取角色列表请求**
   - 构造REQ_GET_ROLE_LIST消息
   - 发送请求到服务器
   - 接收并解析角色列表响应
   - 验证是否获取到角色列表

4. **发送选择角色请求**
   - 构造REQ_SELECT_ROLE消息
   - 发送请求到服务器
   - 接收并解析选择角色响应
   - 验证是否成功选择角色

5. **发送获取成就详情请求**
   - 构造REQ_GET_ACHIEVEMENT_DETAIL消息
   - 设置成就ID为测试数据中的achievementId
   - 发送请求到服务器
   - 接收并解析成就详情响应

6. **验证成就详情数据**
   - 验证响应是否包含成就详情
   - 验证成就详情中的字段是否完整
   - 验证成就ID是否与请求一致
   - 验证成就状态、进度、奖励等信息是否正确

7. **断开连接**
   - 关闭与服务器的TCP连接

## 预期结果

1. 服务器返回成功响应
2. 响应中包含完整的成就详情
3. 成就详情包含以下字段：
   - 成就ID
   - 成就名称
   - 成就描述
   - 成就类型
   - 成就难度
   - 成就状态（已完成/未完成）
   - 当前进度
   - 目标进度
   - 完成时间（如果已完成）
   - 奖励列表
   - 前置成就
   - 后续成就

## 测试异常处理

1. **服务器未运行**
   - 预期：连接失败，测试终止
   - 处理：启动服务器后重新测试

2. **登录失败**
   - 预期：登录响应失败，测试终止
   - 处理：检查登录参数后重新测试

3. **选择角色失败**
   - 预期：选择角色响应失败，测试终止
   - 处理：检查角色GUID后重新测试

4. **成就ID不存在**
   - 预期：响应失败，提示成就不存在
   - 处理：使用正确的成就ID重新测试

5. **获取成就详情超时**
   - 预期：超时异常，测试失败
   - 处理：检查网络连接后重新测试
