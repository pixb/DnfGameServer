# TC009_成就条件验证

## 测试用例信息

- **测试用例ID**: TC009
- **测试用例名称**: 成就条件验证
- **测试类型**: 功能测试
- **优先级**: 中
- **关联模块**: 成就系统
- **测试环境**:
  - JDK 17+
  - Maven 3.6+
  - MySQL 8.0+
  - 游戏服务器运行中

## 测试目标

验证成就系统能够正确验证成就的解锁条件，包括前置成就要求、等级要求、时间要求等各种条件类型。

## 测试前置条件

1. 游戏服务器已启动并正常运行
2. 玩家已成功登录游戏
3. 玩家已选择角色进入游戏
4. 玩家有需要验证条件的成就

## 测试数据

| 字段名 | 类型 | 值 | 说明 |
|-------|------|-----|------|
| openid | String | test_openid_009 | 测试用OpenID |
| version | String | 1.0.0 | 客户端版本号 |
| clientIP | String | 127.0.0.1 | 客户端IP地址 |
| roleGuid | String | role_guid_009 | 角色GUID |
| achievementId | int | 1005 | 测试用成就ID |
| requiredLevel | int | 10 | 要求等级 |
| requiredAchievements | int[] | [1001, 1002] | 要求前置成就 |

## 测试步骤

1. **连接服务器**
   - 建立与游戏服务器的TCP连接
   - 验证连接是否成功

2. **发送登录请求**
   - 构造REQ_LOGIN消息
   - 发送登录请求到服务器
   - 接收并解析登录响应
   - 验证登录是否成功

3. **发送获取角色列表请求**
   - 构造REQ_GET_ROLE_LIST消息
   - 发送请求到服务器
   - 接收并解析角色列表响应
   - 验证是否获取到角色列表

4. **发送选择角色请求**
   - 构造REQ_SELECT_ROLE消息
   - 发送请求到服务器
   - 接收并解析选择角色响应
   - 验证是否成功选择角色

5. **发送成就条件验证请求**
   - 构造REQ_VALIDATE_ACHIEVEMENT_CONDITIONS消息
   - 设置成就ID为测试数据中的achievementId
   - 发送请求到服务器
   - 接收并解析成就条件验证响应

6. **验证成就条件验证响应**
   - 验证响应是否成功
   - 验证条件验证结果是否正确
   - 验证未满足的条件是否明确

7. **验证具体条件验证结果**
   - 验证等级条件是否正确验证
   - 验证前置成就条件是否正确验证
   - 验证其他条件是否正确验证

8. **断开连接**
   - 关闭与服务器的TCP连接

## 预期结果

1. 服务器返回成就条件验证响应
2. 条件验证结果正确反映玩家当前状态
3. 未满足的条件被明确指出
4. 验证结果包含：
   - 是否满足所有条件
   - 未满足的条件列表
   - 每个条件的当前状态
   - 每个条件的目标状态

## 测试异常处理

1. **服务器未运行**
   - 预期：连接失败，测试终止
   - 处理：启动服务器后重新测试

2. **登录失败**
   - 预期：登录响应失败，测试终止
   - 处理：检查登录参数后重新测试

3. **选择角色失败**
   - 预期：选择角色响应失败，测试终止
   - 处理：检查角色GUID后重新测试

4. **成就ID不存在**
   - 预期：响应失败，提示成就不存在
   - 处理：使用正确的成就ID重新测试

5. **成就条件验证失败**
   - 预期：响应失败，测试失败
   - 处理：检查服务器实现后重新测试
