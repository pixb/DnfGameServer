# TC007_成就统计信息

## 测试用例信息

- **测试用例ID**: TC007
- **测试用例名称**: 成就统计信息
- **测试类型**: 功能测试
- **优先级**: 中
- **关联模块**: 成就系统
- **测试环境**:
  - JDK 17+
  - Maven 3.6+
  - MySQL 8.0+
  - 游戏服务器运行中

## 测试目标

验证玩家能够成功获取成就系统的统计信息，包括已完成成就数量、总成就数量、成就完成率等。

## 测试前置条件

1. 游戏服务器已启动并正常运行
2. 玩家已成功登录游戏
3. 玩家已选择角色进入游戏
4. 玩家已有成就数据

## 测试数据

| 字段名 | 类型 | 值 | 说明 |
|-------|------|-----|------|
| openid | String | test_openid_007 | 测试用OpenID |
| version | String | 1.0.0 | 客户端版本号 |
| clientIP | String | 127.0.0.1 | 客户端IP地址 |
| roleGuid | String | role_guid_007 | 角色GUID |

## 测试步骤

1. **连接服务器**
   - 建立与游戏服务器的TCP连接
   - 验证连接是否成功

2. **发送登录请求**
   - 构造REQ_LOGIN消息
   - 发送登录请求到服务器
   - 接收并解析登录响应
   - 验证登录是否成功

3. **发送获取角色列表请求**
   - 构造REQ_GET_ROLE_LIST消息
   - 发送请求到服务器
   - 接收并解析角色列表响应
   - 验证是否获取到角色列表

4. **发送选择角色请求**
   - 构造REQ_SELECT_ROLE消息
   - 发送请求到服务器
   - 接收并解析选择角色响应
   - 验证是否成功选择角色

5. **发送获取成就统计信息请求**
   - 构造REQ_GET_ACHIEVEMENT_STATISTICS消息
   - 发送请求到服务器
   - 接收并解析获取成就统计信息响应

6. **验证成就统计信息响应**
   - 验证响应是否成功
   - 验证统计信息是否完整
   - 验证统计数据是否合理

7. **验证具体统计数据**
   - 验证已完成成就数量是否正确
   - 验证总成就数量是否正确
   - 验证成就完成率是否正确
   - 验证其他统计数据是否合理

8. **断开连接**
   - 关闭与服务器的TCP连接

## 预期结果

1. 服务器返回成功响应
2. 获取成就统计信息成功
3. 返回的统计信息完整且合理
4. 统计数据包括：
   - 已完成成就数量
   - 总成就数量
   - 成就完成率
   - 最近完成的成就
   - 成就点数
   - 成就等级（如果有）
   - 各类别成就完成情况

## 测试异常处理

1. **服务器未运行**
   - 预期：连接失败，测试终止
   - 处理：启动服务器后重新测试

2. **登录失败**
   - 预期：登录响应失败，测试终止
   - 处理：检查登录参数后重新测试

3. **选择角色失败**
   - 预期：选择角色响应失败，测试终止
   - 处理：检查角色GUID后重新测试

4. **获取成就统计信息超时**
   - 预期：超时异常，测试失败
   - 处理：检查网络连接后重新测试

5. **获取成就统计信息失败**
   - 预期：响应失败，测试失败
   - 处理：检查服务器日志后重新测试

## 测试后清理
