# TC006_批量获取成就

## 测试用例信息

- **测试用例ID**: TC006
- **测试用例名称**: 批量获取成就
- **测试类型**: 功能测试
- **优先级**: 中
- **关联模块**: 成就系统
- **测试环境**:
  - JDK 17+
  - Maven 3.6+
  - MySQL 8.0+
  - 游戏服务器运行中

## 测试目标

验证玩家能够成功批量获取多个成就的信息，提高获取成就数据的效率。

## 测试前置条件

1. 游戏服务器已启动并正常运行
2. 玩家已成功登录游戏
3. 玩家已选择角色进入游戏
4. 玩家已有多个成就数据

## 测试数据

| 字段名 | 类型 | 值 | 说明 |
|-------|------|-----|------|
| openid | String | test_openid_006 | 测试用OpenID |
| version | String | 1.0.0 | 客户端版本号 |
| clientIP | String | 127.0.0.1 | 客户端IP地址 |
| roleGuid | String | role_guid_006 | 角色GUID |
| achievementIds | int[] | [1001, 1002, 1003] | 测试用成就ID列表 |
| pageSize | int | 10 | 每页数量 |
| pageNum | int | 1 | 页码 |

## 测试步骤

1. **连接服务器**
   - 建立与游戏服务器的TCP连接
   - 验证连接是否成功

2. **发送登录请求**
   - 构造REQ_LOGIN消息
   - 发送登录请求到服务器
   - 接收并解析登录响应
   - 验证登录是否成功

3. **发送获取角色列表请求**
   - 构造REQ_GET_ROLE_LIST消息
   - 发送请求到服务器
   - 接收并解析角色列表响应
   - 验证是否获取到角色列表

4. **发送选择角色请求**
   - 构造REQ_SELECT_ROLE消息
   - 发送请求到服务器
   - 接收并解析选择角色响应
   - 验证是否成功选择角色

5. **发送批量获取成就请求**
   - 构造REQ_BATCH_GET_ACHIEVEMENTS消息
   - 设置成就ID列表为测试数据中的achievementIds
   - 设置分页参数（pageSize和pageNum）
   - 发送请求到服务器
   - 接收并解析批量获取成就响应

6. **验证批量获取成就响应**
   - 验证响应是否成功
   - 验证是否返回了所有请求的成就信息
   - 验证成就信息是否完整
   - 验证分页信息是否正确

7. **验证每个成就的信息**
   - 遍历返回的成就列表
   - 验证每个成就的ID是否在请求的列表中
   - 验证每个成就的字段是否完整
   - 验证每个成就的状态和进度信息是否正确

8. **断开连接**
   - 关闭与服务器的TCP连接

## 预期结果

1. 服务器返回成功响应
2. 批量获取成就成功
3. 返回的成就列表包含所有请求的成就信息
4. 每个成就的信息完整且正确
5. 分页信息正确（如果使用了分页）

## 测试异常处理

1. **服务器未运行**
   - 预期：连接失败，测试终止
   - 处理：启动服务器后重新测试

2. **登录失败**
   - 预期：登录响应失败，测试终止
   - 处理：检查登录参数后重新测试

3. **选择角色失败**
   - 预期：选择角色响应失败，测试终止
   - 处理：检查角色GUID后重新测试

4. **成就ID列表为空**
   - 预期：响应失败，提示成就ID列表为空
   - 处理：提供有效的成就ID列表重新测试

5. **成就ID不存在**
   - 预期：返回存在的成就信息，忽略不存在的成就ID
   - 处理：使用正确的成就ID列表重新测试

6. **批量获取成就超时**
   - 预期：超时异常，测试失败
   - 处理：检查网络连接后重新测试
