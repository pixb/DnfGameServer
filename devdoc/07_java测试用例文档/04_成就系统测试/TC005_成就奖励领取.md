# TC005_成就奖励领取

## 测试用例信息

- **测试用例ID**: TC005
- **测试用例名称**: 成就奖励领取
- **测试类型**: 功能测试
- **优先级**: 高
- **关联模块**: 成就系统
- **测试环境**:
  - JDK 17+
  - Maven 3.6+
  - MySQL 8.0+
  - 游戏服务器运行中

## 测试目标

验证玩家能够成功领取已完成成就的奖励，包括物品奖励、经验奖励和货币奖励等。

## 测试前置条件

1. 游戏服务器已启动并正常运行
2. 玩家已成功登录游戏
3. 玩家已选择角色进入游戏
4. 玩家已有已完成但未领取奖励的成就

## 测试数据

| 字段名 | 类型 | 值 | 说明 |
|-------|------|-----|------|
| openid | String | test_openid_005 | 测试用OpenID |
| version | String | 1.0.0 | 客户端版本号 |
| clientIP | String | 127.0.0.1 | 客户端IP地址 |
| roleGuid | String | role_guid_005 | 角色GUID |
| achievementId | int | 1003 | 测试用成就ID |
| rewardId | int | 5001 | 测试用奖励ID |

## 测试步骤

1. **连接服务器**
   - 建立与游戏服务器的TCP连接
   - 验证连接是否成功

2. **发送登录请求**
   - 构造REQ_LOGIN消息
   - 发送登录请求到服务器
   - 接收并解析登录响应
   - 验证登录是否成功

3. **发送获取角色列表请求**
   - 构造REQ_GET_ROLE_LIST消息
   - 发送请求到服务器
   - 接收并解析角色列表响应
   - 验证是否获取到角色列表

4. **发送选择角色请求**
   - 构造REQ_SELECT_ROLE消息
   - 发送请求到服务器
   - 接收并解析选择角色响应
   - 验证是否成功选择角色

5. **发送成就奖励领取请求**
   - 构造REQ_CLAIM_ACHIEVEMENT_REWARD消息
   - 设置成就ID为测试数据中的achievementId
   - 设置奖励ID为测试数据中的rewardId
   - 发送请求到服务器
   - 接收并解析成就奖励领取响应

6. **验证成就奖励领取响应**
   - 验证响应是否成功
   - 验证奖励是否已领取
   - 验证奖励领取状态是否更新

7. **发送获取成就详情请求**
   - 构造REQ_GET_ACHIEVEMENT_DETAIL消息
   - 设置成就ID为测试数据中的achievementId
   - 发送请求到服务器
   - 接收并解析成就详情响应

8. **验证成就详情中的奖励状态**
   - 验证成就详情中的奖励领取状态是否已更新
   - 验证奖励是否标记为已领取

9. **断开连接**
   - 关闭与服务器的TCP连接

## 预期结果

1. 服务器返回成功响应
2. 成就奖励领取成功
3. 奖励领取状态更新为已领取
4. 玩家获得相应的奖励（物品、经验、货币等）
5. 成就详情中的奖励状态显示为已领取

## 测试异常处理

1. **服务器未运行**
   - 预期：连接失败，测试终止
   - 处理：启动服务器后重新测试

2. **登录失败**
   - 预期：登录响应失败，测试终止
   - 处理：检查登录参数后重新测试

3. **选择角色失败**
   - 预期：选择角色响应失败，测试终止
   - 处理：检查角色GUID后重新测试

4. **成就ID不存在**
   - 预期：响应失败，提示成就不存在
   - 处理：使用正确的成就ID重新测试

5. **成就未完成**
   - 预期：响应失败，提示成就未完成
   - 处理：完成成就后重新测试

6. **奖励已领取**
   - 预期：响应失败，提示奖励已领取
   - 处理：使用未领取奖励的成就重新测试

7. **奖励ID无效**
   - 预期：响应失败，提示奖励ID无效
   - 处理：使用有效的奖励ID重新测试

8. **成就奖励领取超时**
   - 预期：超时异常，测试失败
   - 处理：检查网络连接后重新测试
