# TC004_成就进度更新

## 测试用例信息

- **测试用例ID**: TC004
- **测试用例名称**: 成就进度更新
- **测试类型**: 功能测试
- **优先级**: 高
- **关联模块**: 成就系统
- **测试环境**:
  - JDK 17+
  - Maven 3.6+
  - MySQL 8.0+
  - 游戏服务器运行中

## 测试目标

验证玩家的成就进度能够正确更新，包括进度增加、进度重置和进度完成的情况。

## 测试前置条件

1. 游戏服务器已启动并正常运行
2. 玩家已成功登录游戏
3. 玩家已选择角色进入游戏
4. 玩家已有未完成的成就

## 测试数据

| 字段名 | 类型 | 值 | 说明 |
|-------|------|-----|------|
| openid | String | test_openid_004 | 测试用OpenID |
| version | String | 1.0.0 | 客户端版本号 |
| clientIP | String | 127.0.0.1 | 客户端IP地址 |
| roleGuid | String | role_guid_004 | 角色GUID |
| achievementId | int | 1002 | 测试用成就ID |
| progress | int | 5 | 测试用成就进度 |
| targetProgress | int | 10 | 成就目标进度 |

## 测试步骤

1. **连接服务器**
   - 建立与游戏服务器的TCP连接
   - 验证连接是否成功

2. **发送登录请求**
   - 构造REQ_LOGIN消息
   - 发送登录请求到服务器
   - 接收并解析登录响应
   - 验证登录是否成功

3. **发送获取角色列表请求**
   - 构造REQ_GET_ROLE_LIST消息
   - 发送请求到服务器
   - 接收并解析角色列表响应
   - 验证是否获取到角色列表

4. **发送选择角色请求**
   - 构造REQ_SELECT_ROLE消息
   - 发送请求到服务器
   - 接收并解析选择角色响应
   - 验证是否成功选择角色

5. **发送成就进度更新请求**
   - 构造REQ_UPDATE_ACHIEVEMENT_PROGRESS消息
   - 设置成就ID为测试数据中的achievementId
   - 设置进度为测试数据中的progress
   - 发送请求到服务器
   - 接收并解析成就进度更新响应

6. **验证成就进度更新响应**
   - 验证响应是否成功
   - 验证成就进度是否已更新
   - 验证成就状态是否正确（如果进度达到目标）

7. **发送获取成就详情请求**
   - 构造REQ_GET_ACHIEVEMENT_DETAIL消息
   - 设置成就ID为测试数据中的achievementId
   - 发送请求到服务器
   - 接收并解析成就详情响应

8. **验证成就详情中的进度**
   - 验证成就详情中的当前进度是否与更新后的进度一致
   - 验证成就状态是否正确

9. **断开连接**
   - 关闭与服务器的TCP连接

## 预期结果

1. 服务器返回成功响应
2. 成就进度更新成功
3. 当进度达到或超过目标进度时，成就状态变为已完成
4. 成就详情中的进度信息与更新后的进度一致

## 测试异常处理

1. **服务器未运行**
   - 预期：连接失败，测试终止
   - 处理：启动服务器后重新测试

2. **登录失败**
   - 预期：登录响应失败，测试终止
   - 处理：检查登录参数后重新测试

3. **选择角色失败**
   - 预期：选择角色响应失败，测试终止
   - 处理：检查角色GUID后重新测试

4. **成就ID不存在**
   - 预期：响应失败，提示成就不存在
   - 处理：使用正确的成就ID重新测试

5. **进度值无效**
   - 预期：响应失败，提示进度值无效
   - 处理：使用有效的进度值重新测试

6. **成就进度更新超时**
   - 预期：超时异常，测试失败
   - 处理：检查网络连接后重新测试
