# Go 服务端开发规划

## 开发目标

将 Java 服务端的所有功能完整迁移到 Go 语言实现，确保功能对等、性能更优、代码更简洁。

## 开发原则

1. **文档先行**：开发功能前先编写设计文档
2. **测试先行**：开发功能前先编写测试用例
3. **参考 Java**：参考 Java 服务端代码实现业务逻辑
4. **渐进开发**：按模块逐步实现，确保每个模块完整可用

## 开发流程

```
1. 分析 Java 代码
   ↓
2. 编写设计文档
   ↓
3. 编写测试用例
   ↓
4. 实现功能代码
   ↓
5. 运行测试验证
   ↓
6. 文档更新
```

## 开发工具

### 代码生成
```bash
# 生成 Protobuf 代码
cd proto
buf generate

# 运行数据库迁移
cd dnf-go-server
make migrate
```

### 测试
```bash
# 运行所有测试
go test ./tests/... -v

# 运行特定测试
go test ./tests -v -run TestAchievementSuite
```

### 构建
```bash
# 构建
./scripts/build.sh

# 运行
./bin/server
```
