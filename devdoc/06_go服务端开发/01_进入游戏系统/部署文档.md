# 进入游戏系统部署文档

## 1. 概述

本文档详细描述了DNF游戏进入游戏系统的部署流程，包括环境准备、依赖安装、配置管理、部署步骤、启动和停止服务等内容。通过本文档，可以指导运维人员快速部署进入游戏系统，确保系统的稳定运行。

### 1.1 文档目的

- 详细描述进入游戏系统的部署流程
- 提供环境准备和依赖安装的指导
- 明确配置管理和部署步骤
- 为运维人员提供参考依据

### 1.2 术语定义

| 术语 | 解释 |
| :--- | :--- |
| 部署 | 将软件从开发环境迁移到生产环境的过程 |
| 环境变量 | 在操作系统中设置的变量，用于配置应用程序 |
| 配置文件 | 包含应用程序配置信息的文件 |
| 依赖 | 应用程序运行所需要的外部库或服务 |
| 服务 | 在后台运行的程序 |
| 容器 | 一种轻量级的虚拟化技术，用于隔离应用程序运行环境 |
| 镜像 | 容器的只读模板，包含应用程序和其依赖 |
| 端口 | 计算机与外部通信的接口 |

## 2. 环境准备

### 2.1 硬件要求

| 配置 | 最低要求 | 推荐要求 |

| 内存 | 4GB | 8GB及以上 |

| 软件 | 版本 | 用途 |

| Redis | 7.0+ | 缓存数据 |

```bash

wget https://golang.org/dl/go1.20.0.linux-amd64.tar.gz

# 解压Go
sudo tar -C /usr/local -xzf go1.20.0.linux-amd64.tar.gz
```

## 4. 配置管理

### 4.1 配置文件结构

```plaintext
config/
├── config.yaml       # 主配置文件
├── config.dev.yaml   # 开发环境配置
├── config.test.yaml  # 测试环境配置
└── config.prod.yaml  # 生产环境配置
```

### 4.2 主配置文件 (`config.yaml`)

```yaml
# 服务器配置
server:
  port: 8080
  host: 0.0.0.0

# 数据库配置
database:
  dsn: "root:password@tcp(localhost:3306)/dnf_game?charset=utf8mb4&parseTime=True&loc=Local"
  max_idle_conns: 10
  max_open_conns: 100

# Redis配置
redis:
  addr: "localhost:6379"
  password: ""
  db: 0

# 认证配置
auth:
  jwt_secret: "your-secret-key"
  jwt_expiration: "24h"
  redis_expiration: "24h"

# 登录配置
login:
  max_login_attempts: 5
  login_cooldown: "1m"
  character_name_pattern: "^[\\u4e00-\\u9fa5a-zA-Z0-9]{2,12}$"
  max_characters_per_account: 8

# 服务器列表配置
server_list:
  cache_ttl: "5m"

# 角色列表配置
character_list:
  cache_ttl: "10m"

# 防作弊配置
anticheat:
  enabled: true
  log_suspicious: true

# 日志配置
log:
  level: "info"
  encoding: "json"

# 监控配置
monitoring:
  enabled: true
  port: 9090
```

### 4.3 环境变量配置

| 环境变量 | 默认值 | 说明 |
| :--- | :--- | :--- |
| `CONFIG_FILE` | `config/config.yaml` | 配置文件路径 |
| `SERVER_PORT` | `8080` | 服务器端口 |
| `DB_DSN` | `root:password@tcp(localhost:3306)/dnf_game` | 数据库连接字符串 |
| `REDIS_ADDR` | `localhost:6379` | Redis地址 |
| `REDIS_PASSWORD` | `` | Redis密码 |
| `REDIS_DB` | `0` | Redis数据库 |
| `JWT_SECRET` | `your-secret-key` | JWT密钥 |
| `JWT_EXPIRATION` | `24h` | JWT过期时间 |
| `LOG_LEVEL` | `info` | 日志级别 |

## 5. 部署步骤

### 5.1 源码部署

#### 5.1.1 克隆代码

```bash
# 克隆代码
git clone https://github.com/your-username/dnf-game-server.git
cd dnf-game-server
```

#### 5.1.2 安装依赖

```bash
# 安装Go依赖
go mod tidy
```

#### 5.1.3 编译代码

```bash
# 编译代码
go build -o bin/login-server cmd/login/main.go
```

#### 5.1.4 初始化数据库

```bash
# 执行数据库初始化脚本
mysql -u root -p < scripts/init.sql
```

#### 5.1.5 配置服务

```bash
# 复制配置文件
cp config/config.yaml config/config.prod.yaml

# 编辑配置文件
vim config/config.prod.yaml
```

#### 5.1.6 启动服务

```bash
# 设置环境变量
export CONFIG_FILE=config/config.prod.yaml

# 启动服务
./bin/login-server
```

### 5.2 容器化部署

#### 5.2.1 编写Dockerfile

```dockerfile
FROM golang:1.20-alpine AS builder

# 设置工作目录
WORKDIR /app

# 复制代码
COPY . .

# 安装依赖
RUN go mod tidy

# 编译代码
RUN go build -o login-server cmd/login/main.go

FROM alpine:latest

# 设置工作目录
WORKDIR /app

# 复制编译产物
COPY --from=builder /app/login-server .
COPY --from=builder /app/config /app/config
COPY --from=builder /app/scripts /app/scripts

# 安装依赖
RUN apk add --no-cache mysql-client

# 设置环境变量
ENV CONFIG_FILE=/app/config/config.yaml

# 暴露端口
EXPOSE 8080
EXPOSE 9090

# 启动服务
CMD ["./login-server"]
```

#### 5.2.2 编写docker-compose.yml

```yaml
version: '3.8'

services:
  login-server:
    build: .
    ports:
      - "8080:8080"
      - "9090:9090"
    environment:
      - CONFIG_FILE=/app/config/config.yaml
      - DB_DSN=root:password@tcp(mysql:3306)/dnf_game
      - REDIS_ADDR=redis:6379
    depends_on:
      - mysql
      - redis
    restart: always

  mysql:
    image: mysql:8.0
    ports:
      - "3306:3306"
    environment:
      - MYSQL_ROOT_PASSWORD=password
      - MYSQL_DATABASE=dnf_game
    volumes:
      - mysql-data:/var/lib/mysql
      - ./scripts/init.sql:/docker-entrypoint-initdb.d/init.sql
    restart: always

  redis:
    image: redis:7.0
    ports:
      - "6379:6379"
    volumes:
      - redis-data:/data
    restart: always

volumes:
  mysql-data:
  redis-data:
```

#### 5.2.3 构建和启动容器

```bash
# 构建容器
docker-compose build

# 启动容器
docker-compose up -d

# 查看容器状态
docker-compose ps
```

### 7.3 常见问题与解决方案

| 问题 | 原因 | 解决方案 |
| :--- | :--- | :--- |
| 服务启动失败 | 端口被占用 | 检查端口占用情况，修改配置文件中的端口 |
| 数据库连接失败 | 数据库服务未启动或连接字符串错误 | 启动数据库服务，检查连接字符串 |
| Redis连接失败 | Redis服务未启动或配置错误 | 启动Redis服务，检查配置文件 |
| 登录失败 | 账号密码错误或账号被封禁 | 检查账号密码，检查账号状态 |
| 服务器列表获取失败 | 服务器状态异常或缓存错误 | 检查服务器状态，清除缓存 |
| 角色创建失败 | 角色名称重复或达到数量限制 | 更换角色名称，删除多余角色 |

## 8. 升级与回滚

### 8.1 升级步骤

#### 8.1.1 源码部署

1. **备份数据**:
   ```bash
   # 备份数据库
   mysqldump -u root -p dnf_game > backup/dnf_game_$(date +%Y%m%d).sql
   
   # 备份配置文件
   cp config/config.prod.yaml backup/config.prod.yaml.$(date +%Y%m%d)
   ```

2. **更新代码**:
   ```bash
   # 拉取最新代码
   git pull
   
   # 安装依赖
   go mod tidy
   
   # 编译代码
   go build -o bin/login-server cmd/login/main.go
   ```

3. **更新数据库**:
   ```bash
   # 执行数据库更新脚本
   mysql -u root -p < scripts/update.sql
   ```

4. **重启服务**:
   ```bash
   # 停止服务
   pkill -f login-server
   
   # 启动服务
   nohup ./bin/login-server > logs/login-server.log 2>&1 &
   ```

#### 8.1.2 容器化部署

1. **备份数据**:
   ```bash
   # 备份数据库
   docker-compose exec mysql mysqldump -u root -p dnf_game > backup/dnf_game_$(date +%Y%m%d).sql
   
   # 备份配置文件
   cp config/config.yaml backup/config.yaml.$(date +%Y%m%d)
   ```

2. **更新代码**:
   ```bash
   # 拉取最新代码
   git pull
   ```

3. **更新容器**:
   ```bash
   # 重新构建容器
   docker-compose build
   
   # 重启容器
   docker-compose up -d
   ```

### 8.2 回滚步骤

#### 8.2.1 源码部署

   ```bash
   pkill -f login-server
   ```


2. **恢复代码**:
   ```bash
   # 切换到之前的版本
   git checkout <commit-hash>
   ```

3. **恢复配置文件**:
   ```bash
   # 恢复配置文件
   cp backup/config.yaml.<date> config/config.yaml
   ```

## 9. 安全加固

### 9.1 网络安全

1. **使用HTTPS**:
   - 配置SSL证书
   - 重定向HTTP到HTTPS

2. **防火墙配置**:
   ```bash
   # 配置防火墙
   sudo firewall-cmd --permanent --add-port=8080/tcp
   sudo firewall-cmd --permanent --add-port=443/tcp
   sudo firewall-cmd --reload
   ```

3. **限制访问IP**:
   ```yaml
   # 限制访问IP
   server:
     allowed_ips: ["192.168.1.0/24"]
   ```

## 10. 故障处理

### 10.1 常见故障

| 故障 | 症状 | 可能原因 | 解决方案 |
| :--- | :--- | :--- | :--- |
| 服务无法启动 | 启动脚本执行后无响应 | 端口被占用 | 检查端口占用情况，修改配置文件中的端口 |
| 数据库连接失败 | 服务启动后报错"database connection failed" | 数据库服务未启动或连接字符串错误 | 启动数据库服务，检查连接字符串 |
| Redis连接失败 | 服务启动后报错"redis connection failed" | Redis服务未启动或配置错误 | 启动Redis服务，检查配置文件 |
| 登录失败 | 客户端登录报错"login failed" | 账号密码错误或账号被封禁 | 检查账号密码，检查账号状态 |
| 服务器列表获取失败 | 客户端获取服务器列表报错 | 服务器状态异常或缓存错误 | 检查服务器状态，清除缓存 |
| 角色创建失败 | 客户端创建角色报错 | 角色名称重复或达到数量限制 | 更换角色名称，删除多余角色 |
| 游戏初始化失败 | 客户端选择角色后报错 | 角色数据损坏或服务端错误 | 检查角色数据，查看服务端日志 |

### 10.3 紧急故障处理

1. **服务崩溃**:
   - 检查日志，定位崩溃原因
   - 重启服务
   - 如无法解决，回滚到之前的稳定版本

2. **数据库故障**:
   - 检查数据库状态
   - 启动数据库服务
   - 如数据库损坏，使用备份恢复

3. **Redis故障**:
   - 检查Redis状态
   - 启动Redis服务
   - 如Redis数据丢失，清除缓存，服务会重新生成数据

4. **网络故障**:
   - 检查网络连接
   - 重启网络服务
   - 如无法解决，联系网络管理员
