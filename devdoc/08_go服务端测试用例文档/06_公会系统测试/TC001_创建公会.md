# TC001_创建公会

## 测试用例概述

- **测试用例ID**: TC001
- **测试用例名称**: 创建公会
- **测试目的**: 验证玩家能够成功创建公会，包括设置公会名称、公会公告等
- **测试类型**: 功能测试

## 前置条件

1. **测试环境要求**:
   - Go服务端已启动
   - SQLite数据库已连接
   - 测试账号已创建

2. **测试数据准备**:
   - 创建测试账号: `test_user_001`
   - 创建测试角色: `TestPlayer1`，等级10
   - 确保角色未加入任何公会
   - 确保公会名称未被使用

3. **依赖服务状态**:
   - 游戏服务器正常运行
   - 数据库连接正常

## 测试步骤

### 步骤1: 构造登录请求
- **操作**: 发送HTTP POST请求到`/api/v1/auth/login`，设置参数:
  - openid: `test_user_001`
  - password: `123456`
  - device: `test_device`
  - clientVersion: `1.0.0`
- **预期结果**: 请求发送成功

### 步骤2: 接收登录响应
- **操作**: 接收服务器返回的登录响应
- **预期结果**: 收到HTTP 200响应

### 步骤3: 验证登录成功
- **操作**: 解析登录响应，验证:
  - error: 0
  - data.token: 非空
- **预期结果**: 登录验证通过

### 步骤4: 构造创建公会请求
- **操作**: 发送HTTP POST请求到`/api/v1/guild/create`，设置参数:
  - name: `测试公会001` (公会名称)
  - announcement: `这是一个测试公会` (公会公告)
- **预期结果**: 请求发送成功

### 步骤5: 接收创建公会响应
- **操作**: 接收服务器返回的响应
- **预期结果**: 收到HTTP 200响应

### 步骤6: 验证创建公会响应
- **操作**: 解析响应，验证:
  - error: 0
  - data.success: true
  - data.guildId: 非空
- **预期结果**: 响应验证通过

### 步骤7: 数据库验证
- **操作**: 查询数据库，验证公会是否已创建，玩家是否成为公会会长
- **预期结果**: 数据库验证通过

### 步骤8: 清理测试数据
- **操作**: 删除测试账号和角色
- **预期结果**: 测试数据清理完成

## 预期结果

- **成功情况**: 公会创建成功，玩家成为公会会长
- **失败情况**: 公会创建失败，返回错误信息
- **边界情况**: 公会名称已被使用，角色已加入公会
