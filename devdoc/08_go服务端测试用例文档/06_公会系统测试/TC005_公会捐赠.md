# TC005_公会捐赠

## 测试用例概述

- **测试用例ID**: TC005
- **测试用例名称**: 公会捐赠
- **测试目的**: 验证玩家能够成功向公会捐赠资源，包括金币、钻石、材料等
- **测试类型**: 功能测试

## 前置条件

1. **测试环境要求**:
   - Go服务端已启动
   - SQLite数据库已连接
   - 测试账号已创建

2. **测试数据准备**:
   - 创建测试账号: `test_user_005`
   - 创建测试角色: `TestPlayer5`，等级10
   - 确保角色已加入目标公会
   - 确保角色有足够的资源进行捐赠

3. **依赖服务状态**:
   - 游戏服务器正常运行
   - 数据库连接正常

## 测试步骤

### 步骤1: 构造登录请求
- **操作**: 发送HTTP POST请求到`/api/v1/auth/login`，设置参数:
  - openid: `test_user_005`
  - password: `123456`
  - device: `test_device`
  - clientVersion: `1.0.0`
- **预期结果**: 请求发送成功

### 步骤2: 接收登录响应
- **操作**: 接收服务器返回的登录响应
- **预期结果**: 收到HTTP 200响应

### 步骤3: 验证登录成功
- **操作**: 解析登录响应，验证:
  - error: 0
  - data.token: 非空
- **预期结果**: 登录验证通过

### 步骤4: 构造公会捐赠请求
- **操作**: 发送HTTP POST请求到`/api/v1/guild/donate`，设置参数:
  - donateType: `gold` (捐赠类型)
  - amount: 10000 (捐赠数量)
- **预期结果**: 请求发送成功

### 步骤5: 接收公会捐赠响应
- **操作**: 接收服务器返回的响应
- **预期结果**: 收到HTTP 200响应

### 步骤6: 验证公会捐赠响应
- **操作**: 解析响应，验证:
  - error: 0
  - data.success: true
- **预期结果**: 响应验证通过

### 步骤7: 数据库验证
- **操作**: 查询数据库，验证角色的资源是否减少，公会的资源是否增加
- **预期结果**: 数据库验证通过

### 步骤8: 清理测试数据
- **操作**: 删除测试账号和角色
- **预期结果**: 测试数据清理完成

## 预期结果

- **成功情况**: 公会捐赠成功，玩家的资源减少，公会的资源增加
- **失败情况**: 公会捐赠失败，返回错误信息
